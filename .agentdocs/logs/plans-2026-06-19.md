# AI·SW 마에스트로 멘토 자동화 에이전트 스킬 스위트 (`asm-mentor-*`)

## Context (왜 만드는가)

사용자는 AI·SW 마에스트로 **멘토**다. swmaestro.ai의 'MY PAGE'는 수동 조작이 불편하고,
**서울/부산 두 개 사이트가 각각 별도 로그인 세션**으로 동작하며 세션이 수시로 만료된다.
멘토의 핵심 반복 업무는 ① **회의실 예약** → ② **멘토링/특강 등록** → ③ 수강 확인 → ④ **보고서 제출**
이며, 화면 간 정보 이동(회의실 시간/장소 → 멘토링 등록 → 보고서 자동 채움)이 많아 수작업 비용이 크다.

목표: `.env` 자격증명으로 서울/부산에 로그인·세션 유지하면서 MY PAGE 9개 영역을 조회/자동화하는
**Claude Code Agent Skill 스위트**를 만들어 배포한다. 별표(★) 기능인
**멘토링/특강 게시판 · 보고 게시판 · 회의실 예약**이 최우선이며, v1은 9개 영역 전체를 구현한다.

### 확정된 사용자 결정
- **브라우저**: 빌드/리콘·실행 모두 **headless**.
- **쓰기 동작**: **명령 시 자동 제출이 기본**. "미리보기/preview" 명시 시에만 제출 직전 정지(`--preview`).
- **v1 범위**: 9개 영역 **풀구현**.
- **스케줄링**: 평소 명령 시 동작. **세션 만료 감지·자동 재로그인 루프는 스케줄 잡에서만** 상시(수동 명령도 ensureSession으로 필요 시 로그인).
- **스킬명**: 전부 **`asm-mentor-` 접두사**로 시작(스위트 구성).
- **구현 방식**: **API-first 하이브리드** — headless 크롬으로 화면 분석/조작도 하되, **네트워크 스캐닝(HAR)으로 내부 API 엔드포인트를 역설계해 직접 HTTP 호출로 빠르게 처리**하고, 브라우저는 로그인·복잡 폼·미매핑/변경 시 폴백으로 사용.

### 리콘으로 확인된 사실 (라이브, 읽기 전용 GET)
- 로그인 DOM은 서울/부산 **동일**: `#username`, `#password`, hidden `csrfToken`(매 로드 갱신), JS `actionLogin()`(`button.btn-login`), `#login_form` → `…/toLogin.do` POST. UTF-8.
- **세션 만료 신호**: 미인증 `…/mypage/…` GET → `302 …/member/user/loginForward.do`.
- **서울·부산이 같은 호스트 `www.swmaestro.ai` + `JSESSIONID; Path=/` 공유** → **region별 별도 컨텍스트 + 별도 storageState 필수**.
- Playwright 1.61.0 + chromium 캐시 + Chrome/Chromium 설치 확인.

## 아키텍처 개요

- **런타임**: Node.js ESM(`.mjs`). 의존성 최소.
  - **HTTP 계층**: Node 내장 `fetch`(undici) — storageState의 `JSESSIONID` 쿠키 재사용 + CSRF 토큰(폼 GET→hidden 추출) 동봉으로 **직접 API 호출**. 가장 빠른 경로.
  - **브라우저 계층**: **Playwright(로컬 `playwright@1.61.0` 고정)**, `channel:"chrome"`(없으면 chromium fallback), `headless:true`, `locale:"ko-KR"`, `timezoneId:"Asia/Seoul"`, viewport 1440×900. 로그인(CSRF+JS), WYSIWYG/파일업로드 폼, 미매핑/드리프트 폴백, 스크린샷에 사용.
- **실행 경로 선택(명령별)**:
  - **조회**(공지/일정/팀/멘토·보고·활동비 목록·뷰/회의실 가용성/회원정보) → **API-first 직접 HTTP**(빠름), 실패 시 브라우저.
  - **쓰기**(멘토링·보고·예약·평가의견) → HAR로 깨끗한 엔드포인트가 확인되면 직접 POST 우선, **CSRF·검증·multipart·WYSIWYG가 얽히면 브라우저** 사용.
  - 로그인/세션/스크린샷 → 항상 브라우저.
- **결정론적 스크립트 vs Claude 추론 분리**: Node 엔진이 모든 네트워크/브라우저 조작 + **stdout JSON 1개**(진단은 stderr)를 소유. SKILL.md(Claude)는 명령/region 선택·의도수집·`--preview` 판단·JSON 해석/한국어 요약·**Notion MCP 교차조회**·파괴적 작업 전 확인을 소유.
- **셀렉터·엔드포인트는 코드가 아니라 데이터**: `references/selectors.json`(`_default`+region 오버라이드)와 `references/endpoints.json`(method/url/params/response/csrf)에 보관. 사이트 변경 시 `recon`으로 재추출 → JSON만 교체(코드 무수정).

## 세션 + HTTP/API 계층

- **저장**: region별 `storageState` JSON, region별 **별도 컨텍스트**. HTTP 계층은 같은 storageState에서 `JSESSIONID`를 읽어 쿠키 헤더로 재사용 → 브라우저/HTTP가 **하나의 세션 공유**.
- **만료 감지**: `isLoginPage = /\/member\/user\/(loginForward|forLogin)\.do/`. HTTP 경로는 302 리다이렉트/로그인 폼 HTML로, 브라우저 경로는 URL로 감지.
- **`gotoGuarded`/`httpGuarded`**: 요청 결과가 로그인 페이지면 `.env`로 **재로그인 후 1회 재시도** → 실패 시 `SESSION_EXPIRED`. 재로그인은 항상 브라우저로 수행(매번 `forLogin.do` 새로 열어 최신 csrfToken 확보, 수제 POST·토큰 캐싱 금지) → 갱신된 storageState를 HTTP 계층도 즉시 사용.
- **CSRF(쓰기)**: 직접 POST 시 대상 폼을 GET해 hidden `csrfToken` 추출 후 동봉. 불확실하면 브라우저 폼 제출로 폴백.
- **스케줄 잡**: cron 레시피(`/schedule`/cron)가 주기적으로 `login --force`/`session-status`로 세션 워밍 + 만료 자동 재로그인(+선택 공지/일정 다이제스트·보고 리마인드). v1엔 문서/레시피로 제공.

## 핵심 워크플로우 (★)

### A) 회의실 예약
- `room-availability`(http): 슬롯 그리드/AJAX에서 `slots:[{start,end,status,reserver,checkboxValue}]`. capacity≥인원 필터. `N=ceil(분/30)` 연속 free 구간 → 선호창/최소충분용량/이른시작 랭크.
- `room-reserve`: 직전 재확인(stale 방지) → 제목/인원/내용·연속 슬롯 → 제출 → `itemRent/list`로 확인, `rentId` 반환. 외부(토즈)는 링크 안내만.

### B) 멘토링/특강 등록 (서울/부산 폼 상이)
- **선행 회의실 확보**(A 재사용 또는 `--reserve-room`) → 그 장소를 진행장소로.
- **양 region 시간충돌 검사**: 서울+부산 내 멘토링 파싱 → 같은 날 `newStart<exist.end && exist.start<newEnd`면 거부(`--force` 예외). 종료시간 없으면 `DEFAULT_DURATION` 추정·명시.
- 폼 의존순: 강의구분 → (부산)진행방식→진행장소 재populate → 모집명 → 접수기간(강의시작전까지/직접입력, date+hh시mm분) → 강의날짜(접수기간 이후) → 수강인원(자유≥2/특강≥6) → 진행장소 → 첨부 → WYSIWYG 본문(에디터 종류 감지 후 `setData`).

### C) 보고서 제출 (가장 복잡)
- **자동채움**: 멘토링 `mento-view`(qustnrSn) → 진행날짜/시간/장소/참여연수생(이름·수)/주제→요약/추진내용 매핑. 멘토링대상=region 기반. 구분=강의구분(정규멘토링은 수동, 이때만 팀명 활성). 참여연수생수=이름 수 자동.
- **중복 방지(2단)**: ① 로컬 원장 `state/reported.json`({region}:{qustnrSn}) ② 라이브 fuzzy(날짜|구분|장소|시작시간 fingerprint로 `report-list` 대조) → 이미 보고면 `--force` 없이 거부.
- **증빙(필수)**: 파일 레퍼런스를 `추가+` 행 추가 후 `setInputFiles`. **선택 시 멘토링 뷰 fullPage 스크린샷 자동 캡처해 증빙 첨부**. 증빙 0개면 거부.
- **수동 오버라이드**: `report:draft`가 `필드|자동값|출처` 표 → 응답 `{action:confirm|edit|cancel, overrides, evidence, autoScreenshot, preview, force}` 병합. override 필드 출처=manual.

### 공용 프리미티브(`widgets.mjs`, A/B/C 공유)
달력 date-set, hh/mm 셀렉트, add-row(참여연수생·팀명·무단불참자·증빙 공통), 네이티브 submit 래퍼, setInputFiles, fullPage 스크린샷.

2. **세션 코어**: `login`/`session-status`/`ensureSession`/`gotoGuarded`/`httpGuarded` → 서울·부산 headless 로그인·재로그인·storageState·**HTTP 쿠키 재사용** 검증.
3. **리콘 + 네트워크 스캐닝**: `recon`로 9개 영역 + 3대 폼 DOM·옵션·hidden·CSRF·에디터·**HAR(AJAX/.do 엔드포인트)** 캡처 → `selectors.json`/`endpoints.json`/`site-notes.md`.
4. **조회 명령(API-first)**: notices/schedule/team/mento·report·fund list·view/room-availability/member-info를 직접 HTTP로, 브라우저 폴백. Notion 교차.
5. **쓰기 명령(★)**: room-reserve/cancel → mento-create/update/delete → report-create → fund-comment. 깨끗한 엔드포인트는 직접 POST, 복잡 폼은 브라우저. `widgets.mjs` 공용화.
6. **스킬 5종 SKILL.md + references**: core(엔진·명령표·JSON계약·실행경로·preview·확인규칙) + room/mentoring/report/board(트리거·코어 위임·워크플로우 A→B→C). `allowed-tools: [Bash + Notion MCP]`.
7. **스케줄 레시피(문서)**: 세션 keep-alive/다이제스트/리마인드 cron 예시.

## 검증 (end-to-end)
- **세션**: 서울·부산 `login`/`session-status`; storageState 삭제 후 명령이 자동 재로그인(`meta.reLoggedIn:true`); HTTP·브라우저가 같은 세션 공유.
- **API-first**: 조회 명령이 `meta.path:"http"`로 빠르게 동작, 엔드포인트 강제 무효화 시 `meta.path:"browser"` 폴백.
- **조회**: 각 list/view 유효 JSON·한글 정상. team↔Notion 매칭.
- **회의실(가역)**: 먼 미래 `room-availability`→`room-reserve`→`itemRent/list`→`room-cancel`.
- **멘토링(가역)**: 테스트 제목 `mento-create`→`mento-view`→`mento-delete`. 서울·부산 충돌검사 동작.
- **보고서(주의)**: 실제 멘토링으로 `report-create --preview`까지(자동채움·증빙·스크린샷·중복감지). 실제 제출은 사용자가 대상 지정 시에만.
- **실패 경로**: 미매핑 시 `ENDPOINT_NOT_FOUND`/`SELECTOR_NOT_FOUND`+recon 힌트, 시크릿 미노출.

## 리스크 / 미지 (라이브 의존)
- 서울·부산 동시 세션의 서버측 상호 무효화 여부.
- 비로그인 외 **모든 셀렉터/엔드포인트/옵션/CSRF/에디터/캐스케이드/add-row name/슬롯 그리드/성공 신호**는 리콘 후 확정.
- 직접 API 쓰기가 서버 검증(parsley/CSRF/세션 바인딩)에 막힐 가능성 → 브라우저 폴백으로 흡수.
- 보고 게시판 Busan 처리(서울 보드 작성) 리콘 재확인. 반복 로그인 레이트리밋/캡차(storageState 재사용·실Chrome 완화).
